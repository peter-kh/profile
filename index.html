Projects

Smart Load Forecasting for University Building

Problem Definition: Energy demand varied by season and schedule, creating inefficient load planning.

Technical Approach: Built a structured consumption baseline and segmented demand by time blocks, occupancy, and weather context.

Tools / Methods Used: Excel modeling, time-series comparison, basic statistical validation, load profiling.

Business Perspective: The model can scale to similar facilities as a low-cost decision-support layer for scheduling and energy budgeting.

Current Status: Prototype

Hybrid Solar-Diesel Feasibility Study

Problem Definition: High fuel dependency increased operational cost and introduced supply risk.

Technical Approach: Compared baseline diesel-only performance against hybrid configurations using demand assumptions and peak-load constraints.

Tools / Methods Used: Energy yield estimation, cost-benefit matrix, sensitivity checks, system sizing logic.

Business Perspective: Feasibility depends on fuel price trends and payback period; strongest case is for sites with stable daytime demand.

Current Status: Development Phase

Energy Monitoring Web Application

Problem Definition: Small teams often lack a simple interface to track usage trends and performance indicators.

Technical Approach: Designing a modular dashboard flow for data entry, visualization, and monthly trend interpretation.

Tools / Methods Used: Frontend web stack, KPI structuring, practical workflow mapping.

Business Perspective: A lightweight subscription model is feasible for small organizations needing basic monitoring without enterprise complexity.

Current Status: Application currently in development.
